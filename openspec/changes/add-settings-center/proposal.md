# Change: Add Settings Center (系统设置中心)

## Why
当前系统仅有零散的设置/退出入口，无法承载账号管理、会员奖励、用量统计、外观字体、最近删除、关于等功能。需要一个统一的“系统设置中心”承载这些能力，并与现有 Dashboard 导航一致。

## What Changes
- 新增 **系统设置中心页面**：左侧设置菜单 + 右侧内容区（参考提供的设计图）。
- 新增设置中心的一级入口（Dashboard 底部齿轮按钮进入）。
- 新增/完善以下模块（可分阶段落地）：
  - 3.1 个人中心：个人信息展示与修改、修改密码、绑定邮箱/手机号/微信、退出登录/切换账号（明显位置）。
  - 3.2 会员奖励：推荐码领取 7 天（一次）、邀请好友赠送（双方 7 天，最高 49 天）、分享推荐码。
  - 3.3 用量统计：加入天数、收藏/收藏夹/智能列表/标签/标注、笔记字数、累计访问次数；类型分布图（饼图/柱状图）；来源 TOP10（收藏/访问）。
  - 3.4 外观主题与字体：主题（系统/浅色/深色）、阅读器自定义字体导入。
  - 3.5 最近删除：回收站列表、恢复/永久删除。
  - 3.6 关于 NewsBox：使用指南、帮助文档、反馈、联系邮箱/微信等。

## Impact
- **Affected specs**:
  - `openspec/specs/dashboard/spec.md`（导航入口扩展）
  - 新增 `settings` capability spec（设置中心能力定义）
- **Affected code (expected)**:
  - `app/settings/*` 或 `app/dashboard/settings/*`（新增页面路由与布局）
  - `components/settings/*`（设置中心 UI）
  - `lib/services/*`（统计/推荐码/最近删除等服务封装）
  - `supabase/migrations/*`（若启用最近删除、推荐码奖励等需新增表/字段）

## Non-Goals (MVP)
- 不在 MVP 阶段接入真实 **微信 OAuth**、短信验证码发送等第三方能力（可先做 UI 入口与占位），除非明确要求本期必须上线。
- 不在 MVP 阶段完成所有统计的实时精确计算（可先按“近似/异步汇总”实现，后续优化为准实时）。

## Open Questions
1. **绑定手机号/微信**：本期是否需要真实接入（短信/微信 OAuth），还是先提供入口与占位？
2. **最近删除**：删除当前是否应改为“软删除 + 回收站（可恢复）”，还是仅记录删除日志？
3. **用量统计**：统计口径是否按“当前有效笔记”还是包含已归档/已删除？“累计访问次数”是否以 `notes.last_accessed_at` 为基础做事件表？
