# Change: Enhance hierarchical folder management

## Why
- 当前“收藏夹”只能创建单层分组，无法建立子分类，导致大量收藏时缺乏层级管理，用户难以根据主题/项目进一步整理内容。
- 缺少显式的收藏夹创建入口和后续管理操作（重命名、图标、归档、删除），用户必须依赖后台或手动数据调整，体验割裂。
- 收藏夹操作缺乏一致的 UI 规范（例如“...”菜单、点击后展示该收藏夹的新闻列表），需要在 Spec 层明确行为以支撑后续实现。

## What Changes
- 规定“收藏夹”区域的 `+` 按钮行为：可创建顶层收藏夹，并在创建时选择父级以支持子分类。
- 要求收藏夹支持嵌套层级（至少 2 级），列表需以缩进/树状结构展示父子关系，并在导航中保持状态同步。
- 定义收藏夹的上下文操作菜单（“...”）和必须实现的动作：重命名、更换图标、创建子收藏夹、归档、删除。
- 明确点击任一收藏夹（含子级）时，主列表必须过滤到该收藏夹下的全部新闻/笔记。
- 扩展数据模型（Supabase `folders` 表）以保存父子关系、图标、归档状态，并补充相应 API/权限要求。

## Impact
- Affected specs: `library`
- Affected code: Supabase `folders` 表/迁移、收藏夹 CRUD API、我的收藏页面侧边栏（数据加载、树展示、操作菜单）、与笔记列表过滤/计数交互。
