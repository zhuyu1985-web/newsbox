# Change: Enhance list filtering and view modes

## Why
- 用户需要更强大的筛选能力来管理大量收藏内容，包括查看已归档项、按内容类型过滤（文章、网页、片段、速记、图片、语音、视频、文件）、以及灵活的排序选项（按创建/更新时间、标题、网站排序）。
- 当前列表仅支持单一卡片视图，用户在不同场景下需要不同的展示密度：详情卡片适合浏览、紧凑卡片/列表适合快速扫描、标题列表适合大规模管理，缺少这些视图会降低使用效率。
- "添加"按钮当前显示文字标签且占用较多空间，需要优化为圆形图标按钮，并将添加网址、速记、上传功能整合到下拉菜单中，提升 UI 简洁性和一致性。

## What Changes
- 在顶部搜索框后方新增工具栏区域，包含"已归档"、"筛选"、"排序"、"视图"四个功能按钮。
- "已归档"按钮用于切换展示已归档/未归档内容。
- "筛选"下拉支持按内容类型过滤：不限、文章、网页、片段、速记、图片、语音、视频、文件。
- "排序"下拉支持 8 种排序模式：按创建时间（从新到旧/从旧到新）、按更新时间（从新到旧/从旧到新）、按标题 A-Z/Z-A、按网站 A-Z/Z-A。
- "视图"下拉支持 5 种展示模式：详情卡片（当前样式）、紧凑卡片、详情列表、紧凑列表、标题列表。
- 顶部"添加"按钮改为圆形图标按钮（移除"添加"文字），点击显示下拉菜单：添加网址、添加速记、上传图片/视频/文件。

## Impact
- Affected specs: `library`
- Affected code: 
  - Dashboard/library UI 顶部工具栏（新增筛选、排序、视图、归档按钮）
  - Notes API 查询参数扩展（支持 archived、content_type filter、sort order）
  - 前端状态管理（筛选、排序、视图模式的持久化）
  - 列表渲染逻辑（根据视图模式切换卡片/列表布局）
  - "添加"按钮 UI 重构为圆形图标 + 下拉菜单
