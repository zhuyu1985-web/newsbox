关键决策

05 · 旗舰子系统

视频处理流水线状态机

Next.js 同进程 setInterval Worker 驱动 video_jobs，调用腾讯 COS CI、阿里听悟、Qwen-VL。错误隔离 · 幂等守卫 · 防腐三重保护 · 原始文件保留。

download→ probe + cover→ transcode→ audio（听悟）→ frame→ visual（Qwen-VL）→ reconcile
processing处理中
→
media_ready媒体可播放
→
fully_ready媒体 + AI 全就绪
⤷　异常分支
need_browser_fallbackdownload 需浏览器兜底 · 优先级最高 · 可插件重传
failed任一 step 失败且 retry 达上限
图 2 · video_overall_status 聚合状态流转（computeOverall）

防腐三重保护

转码 done 时：① 守卫 transcoded_key 存在 → ② exists() HEAD 校验产物 → ③ 只写 transcoded_url，绝不覆盖 cos_key，防 CI 假成功导致原始文件不可逆丢失。

关键设计

visual 失败 / skipped 永不阻塞 fully_ready；audio 必须等 transcode 终态（DASH 分轨合流后才有音轨）；403 → need_browser_fallback。

⚠️ 已知风险

多实例无强锁（重复消费）；retry_count 为 job 级全局计数；audio_status CHECK 缺 skipped 但代码会写。

08 · 风险与改进

优先收口的关键风险

完整清单见技术说明文档 §7（按安全 / 一致性 / 性能 / 架构分组，含文件位置与修复建议）。

🔴 视频 AI 全链路无会员门禁

任何登录用户（含试用过期）可触发下载→转码→听悟→Qwen-VL 最烧钱链路。影响成本与商业模式，建议统一补 requireAIMembership。

🔴 支付回调幂等无行锁

并发重复回调理论上可双发会员。建议改 UPDATE ... WHERE status='pending' 原子 CAS。

🟡 阅读页双重大查询

Server 查完丢弃、Client 再查一遍；initialNote 传递为死代码待接。

🟡 巨型组件 + Worker 多实例

dashboard-content.tsx ~6000 行；视频 Worker 多副本无锁会重复消费，建议单实例或独立 worker。

🟡 确认缺陷

tags/reorder 漏 select parent_id；audio_status CHECK 缺 skipped；npm test 当前 23/260 失败（多为陈旧测试）。

⚪ 调试残留

ai/read、knowledge/chat、DashboardAuthCheck 处残留 console.log（含 key 元信息/会员 JSON），上线前清理。
